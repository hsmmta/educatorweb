## 第三页：团队分工

### 参考 PPT 结构
三人分工，每人包含：负责方向 + 具体技术点 + 协调接口

### 本系统仿写

**页面标题**：团队分工 — 三大技术方向协同推进

---

**成员一：多智能体编排与后端架构**

- **主要负责**：整体架构协调、多智能体 DAG 编排引擎、后端核心服务

- 统筹赛题功能性验证（多智能体≥5类资源生成、画像≥6维、防幻觉命中率等）
- 协调各模块接口联调（画像↔资源生成↔路径规划↔辅导↔评估的数据流）
- 自研 `GraphOrchestrator` DAG 编排引擎（五元组模型：节点/边/扇出/路由/重试），支持并行扇出、条件路由与 Redis 断点续跑
- 实现 `ReviewAgent` L1/L2 双重质检机制与最多 3 次重试策略
- 设计 `GenerationState` 不可变状态传递与 `ModelRegistry` 多厂商 LLM 路由

- **与 AI 模块协同定义**：`AgentNode` 生成器接口规范、`ProgressEvent`/`StreamEvent` SSE 事件格式
- **与前端模块协同定义**：`POST /api/generate` 资源生成接口、`POST /api/tutor/chat/stream` 流式问答接口

---

**成员二：大模型集成与知识库构建**

- **主要负责**：LLM 调用架构、7 类资源生成 Agent、知识图谱与向量数据库

- 实现 `DeepSeekProvider`（Spring AI `ChatClient` 流式调用）及 LangChain4j `ProfileUpdateAgent` 声明式 Agent
- 设计 7 类生成器（DOC/PPT/QUIZ/CODE/HTML/MINDMAP/VIDEO）的专用 Prompt 模板与个性化参数注入
- 构建 Neo4j 知识图谱：22 门机器学习课程、20 个静态学习资源、知识点先修/关联关系，实现 `KgBuildAgent` 四阶段构建算法与 LLM 兜底更新
- 搭建 Qdrant 文档向量检索（私有智库 RAG）+ Chroma 对话记忆引擎（滑动窗口分层记忆）
- 设计六维画像 LLM 提取模板与 `sanitizeProfileForDb()` 中文→DB 枚举归一化逻辑
- 实现艾宾浩斯遗忘衰减算法（halfLife 五级分级，3-30 天）与三维掌握度评估

- **与后端模块协同定义**：LLM Provider 接口（`chat()` + `chatStream()`）、RAG 检索接口（userId 隔离 + 相关性阈值）
- **与前端模块协同定义**：资源生成 SSE 事件的 `payload` 数据结构、Markdown/LaTeX/Mermaid 渲染格式

---

**成员三：前端界面与交互体验**

- **主要负责**：Vue 3 SPA 全页面开发、流式渲染与交互组件、AI 语音助手

- 完成全部 10 个页面开发（Chat/Learning/Profile/Push/Tutoring/ThinkTank/ProfileChat/EditProfile/Login/Register）
- 实现 SSE 流式双通道：资源生成进度（`ProgressEvent` 实时渲染 Agent 流程动画）+ AI 问答 token 流（`StreamEvent` 逐字追加 Markdown 渲染）
- 开发交互式组件：题库答题卡片（MC/TF 即时对错反馈）、代码编辑器（在线运行+输出面板）、思维导图（Mermaid 异步渲染 SVG）、HTML 课件 iframe 沙箱
- 集成多模态渲染引擎：marked.js（Markdown）+ KaTeX（LaTeX 公式）+ Mermaid（流程图）
- 实现 Live2D 虚拟角色 + Three.js 3D 文具装饰 + 浏览器 Web Speech API 语音交互
- 设计响应式布局与 Element Plus 统一 UI 规范（紫色渐变主色 #667eea→#764ba2，卡片化设计）

- **与后端模块协同定义**：SSE 事件解析（`EventSource` + `ReadableStream`）、REST API 请求/响应格式
- **与 AI 模块协同定义**：前端渲染所需的 Markdown/JSON/Mermaid 格式规范、交互组件数据结构

| # | 核心挑战 | → | 技术方案 | 技术落地 |
|:---:|---------|:---:|---------|---------|
| ① | **学情感知能力缺失，大模型交互千人一面**：传统系统无长期学生画像与跨会话记忆，问答、出题、评估各自孤立，无法适配个体差异 | → | **多源异构学情画像引擎**：实现"对话—行为—知识"三通道学情量化解析 | `ProfileExtractionAgent` + `PassiveProfileUpdateService` |
| ② | **资源生成孤立低效，质量控制与知识溯源缺失**：单次LLM仅产一种资源，无并行、无审核、无溯源，七类资源串行耗时分钟级 | → | **多智能体DAG协同生成**：实现"知识图谱×大模型×质检审核"三引擎并行资源产出 | `GraphOrchestrator` + `ReviewAgent` |
| ③ | **知识状态追踪缺失，学习评估止于静态统计**：传统正确率无法区分"昨天答对"与"半年前答对"，遗忘规律与评估可信度双重缺失 | → | **遗忘感知的动态知识追踪**：构建"正确率×遗忘曲线×置信度"三维学情量化评估 | `ProficiencyService` + `LearningReportService` |
