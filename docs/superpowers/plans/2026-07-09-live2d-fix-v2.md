# Live2D PIXI 6.2.2 兼容 + 清推送历史 + 资源跳转修复

> **根因:** `pixi-live2d-display@0.4.0` 使用 PIXI 6.0-6.3 的旧 `renderer.plugins.interaction` API，6.4+ 用 `renderer.events` 替换了它。装 6.5.10 导致 `manager.on is not a function` / `currentTarget.isInteractive is not a function`。
> **修复:** pixi.js@6.5.10 → pixi.js@6.2.2（最后支持旧交互 API 的版本）。

## 子代理任务

### Task 1: 安装 PIXI 6.2.2

```bash
cd "E:/educatorweb/educatorweb/frontend"
npm install pixi.js@6.2.2
```

**验证:** `npm --prefix frontend run build` 无误

### Task 2: 清理推送历史（可选——后端）

在 MySQL 中删除 push_result 表的旧记录，让用户从干净状态重新测试推送。

```sql
DELETE FROM push_result WHERE user_id = '18551156503';
DELETE FROM topic_cache WHERE user_id = '18551156503' AND pushed = true;
```

或在后端临时加一个接口 `DELETE /api/push/history?studentId=xxx`

### Task 3: 资源跳转回 /learning

将 ResourcePush.vue 的 `goLearn` 从 `/chat` 改回 `/learning?topic=xxx`。因为 `/chat` 不会生成资源，`/learning` 的多智能体管道才是真正生成资源内容的地方。
