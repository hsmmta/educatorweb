# 资源推送页面重新设计

> **日期:** 2026-07-09
> **分支:** resource-push
> **背景:** 当前 ResourcePush.vue 存在三个问题:打开后需要手动输入知识点才能看到内容;"今日学习"选项卡指向旧版沉浸学习页;页面功能与后端实际能力(话题推送/学习路径/资源推荐/SSE 通知)不匹配。本次重新设计以**后端实际能力为准**,PPT 和需求文档之后再改。

## 1. 目标

将 `/push` 页面从"需要输入才能使用的工具页"改造为**以"今日推送"为主的个性化资源入口页**。用户打开页面第一眼看到系统推送的内容,同时保留自主探索知识点的能力。

## 2. 页面整体布局

```
┌──────────────────────────────────────────────────────────────────┐
│  📬 资源推送                                          [刷新]      │
├─────────────────────────────┬────────────────────────────────────┤
│  📬 系统推送 (Scheduled)    │  🔍 自主探索                        │
│                             │                                    │
│  📐 线性回归                │  [输入知识点...]          [规划]     │
│    [📄课程] [📝练习] [💻代码] │                                    │
│                             │  📋 最近学过                        │
│  📉 梯度下降                │  [线性回归] [SVM] [决策树]           │
│    [🎬视频] [🧩导图]        │                                    │
│                             │  ⚠️ 需要巩固                        │
│  展开 +3 个话题 ▾           │  [过拟合 32%] [特征选择 45%]        │
│                             │                                    │
├─────────────────────────────┴────────────────────────────────────┤
│  [底部弹出面板 — 默认不显示]                                        │
│  搜索结果: 学习路径时间线 · 推荐资源卡片                              │
│  推送历史: 按时间倒序的推送记录列表 · 选中记录的资源详情              │
└──────────────────────────────────────────────────────────────────┘
```

### 2.1 左卡片:系统推送

**数据来源:** `GET /api/push/latest?studentId=xxx`(即 `PushResultController.getLatest`,返回最新推送记录中 `resources` JSON 里的每个话题 + 资源列表)。

**展示逻辑:**
- 卡片标题显示推送触发类型:`📬 今日推送 · COUNT 触发` 或 `📬 每日推送 · SCHEDULED 触发`
- 无推送时显示:`📭 暂无推送 — 继续对话学习,系统会自动为你整理话题`
- **默认展开前 2 个话题**,每个话题展示:
  - 话题名称(弱点是🔴红色,正常是蓝色)
  - 具体资源列表(每个资源是类型图标 + 类型名的小标签:📄课程文档、📝练习题库、💻代码案例、🎬视频讲解、🧩思维导图、🌐交互页面)
  - 点资源标签跳转到资源(如果是 HTML 类型用新窗口 `/learning?topic=xxx`,否则在当前页打开)
- 第 3 个及以上话题折叠,底部显示 `展开 +N 个话题 ▾`,点击后内联展开所有话题,底部变为 `收起 ▲`
- 展开区域设 `max-height: 320px; overflow-y: auto`,防止卡片过高推出右卡片
- 卡片底部有 `查看推送历史 →` 链接,点击触发底部面板加载推送历史

### 2.2 右卡片:自主探索

**两个状态:初始态(推荐)+ 结果态(不主动展示,搜索结果走底部面板)**

**初始态:**
- 搜索框 + `规划` 按钮(按回车也可触发)
- 搜索框有自动补全:输入时调用后端接口,候选列表为 Neo4j 知识图谱节点名 + TopicCache 话题名合并去重,用户自己的话题排前面
- 如果有会话历史:`📋 最近学过` 一行标签,从 TopicCache 表取最近 5 个话题。点击标签自动填入搜索框
- 如果有薄弱数据:`⚠️ 需要巩固` 一行标签,从 StudentKnowledgeProficiency 取 `< 0.6` 的前 5 个知识点,附带熟练度百分比。点标签自动填入搜索框
- 如果都没有(新用户):只显示搜索框 + 引导文案 "输入你想学的知识点,系统为你规划个性化学习路径"

### 2.3 底部弹出面板

**两种内容模式,共用同一个面板区域:**

**模式 1: 搜索结果**(用户点搜索框旁的"规划"或按回车触发)
- 面板从下方滑入
- 调用 `GET /api/push/recommend/{studentId}?target=xxx`
- 展示内容:
  - 左栏:学习路径时间线(节点名、难度星星、状态标记 COMPLETED/CURRENT/PENDING,当前节点高亮)
  - 中栏:路径概览(总节点数、已完成数、预计天数)
  - 右栏:推荐资源列表(从 allRecommendations 取,每个资源一行:图标+标题+类型+理由),点击资源可跳转学习
- 面板右上角有 ✕ 关闭按钮
- 面板内容区域 `max-height: 400px; overflow-y: auto`

**模式 2: 推送历史**(用户点左卡片底部"查看推送历史 →"触发)
- 面板切换为列表视图
- 调用 `GET /api/push/results?studentId=xxx`
- 左侧:推送记录列表(时间倒序,每条显示触发类型标签 + 时间 + 资源数量)
- 右侧:选中记录的详情(按话题分组,每组显示话题名 + 是否为薄弱点 + 该话题的具体资源标签)
- 点击资源标签可跳转学习

### 2.4 资源"学习"跳转

- HTML 类型的资源:在新窗口打开(资源本身是 HTML 文件时)
- 其他类型:跳转到 `/learning?topic=xxx`(旧版沉浸学习页带主题参数),用户可以在那里继续使用资源生成功能
- 不做当前页面内 iframe 嵌入(保持页面交互简洁)

## 3. 首页改动

`/home` 页面当前只有"智能辅导"和"个人中心"两个按钮。改为三个按钮:

```
智能辅导 → /chat
资源推送 → /push  (新增)
个人中心 → /profile
```

- 按钮风格保持与现有 DeepSeek 风格一致(圆角、渐变)
- `/learning`(旧版沉浸学习页)路由保留但不从首页链接,仅在资源页的"学习"跳转中作为目标页使用

## 4. SSE 实时通知集成

当前 MainLayout 已有 SSE 订阅(`subscribePushApi`)和铃铛 badge。保留现有机制,在此基础上:

- 当 SSE 推送新通知时,铃铛 badge 计数 + 1,同时如果用户在 `/push` 页面,自动刷新左卡片的推送内容(调 `getLatestPushApi`)
- 铃铛点击仍跳转到 `/push`

## 5. 后端需新增的 API

| 接口 | 用途 | 说明 |
|------|------|------|
| `GET /api/push/autocomplete?studentId=xxx&q=支` | 搜索自动补全 | 返回 Neo4j 节点名 + TopicCache 话题合并去重,前 10 条 |
| `GET /api/push/recent?studentId=xxx` | 最近学过的知识点 | 从 TopicCache 取该用户最近 5 个话题 |
| `GET /api/push/weakness?studentId=xxx` | 薄弱知识点 | 从 StudentKnowledgeProficiency 取 < 0.6 的前 5 个 |

或可合并为一个 `GET /api/push/context?studentId=xxx` 返回 `{ recentTopics: [...], weaknessTopics: [...] }`。

## 6. 前端文件改动

| 文件 | 操作 | 说明 |
|------|------|------|
| `frontend/src/views/ResourcePush.vue` | **重写** | 完全替换为新的双卡片 + 底部面板布局 |
| `frontend/src/views/Home.vue` | **修改** | 两按钮 → 三按钮,加"资源推送"入口 |
| `frontend/src/views/MainLayout.vue` | **修改** | SSE 收到通知时,若当前在 /push 则自动刷新 |
| `frontend/src/api/index.js` | **修改** | 新增 autocomplete/recent/weakness 等 API 函数 |

## 7. 不做的事(YAGNI)

- 不引入新的前端状态管理(pinia)——页面级 ref + computed 够用
- 不改造 `/learning`(旧沉浸学习页)——保留路由,仅作为资源"学习"跳转目标
- 不引入 websocket 替代 SSE(SSE 已工作)
- 不修改 PPT/需求文档(之后改)

## 8. 验证方式

1. 首页三个按钮可点击,资源推送按钮跳转 `/push`
2. 有推送数据时左卡片展示具体资源,无推送时显示空状态提示
3. 话题数 ≤2 时无折叠;>2 时折叠 + "展开 +N"正常收起和展开
4. 搜索 "支" 自动补全列出相关知识点
5. 点击"最近学过"/"需要巩固"标签自动填入搜索框
6. 点击搜索框旁的"规划"或按回车,底部面板弹出学习路径 + 推荐资源
7. 点击"查看推送历史 →",底部面板切换到推送历史列表
8. 底部面板可以 ✕ 关闭
9. SSE 新推送到来时铃铛 badge 更新, `/push` 页面左卡片自动刷新
10. 点击资源标签跳转到正确的学习页面
